I'm trying to create a command list file that employs batchlists for some commands. It is working so far except for one thing. I'm creating a list of HiRISE BG/IR ccd's, then using each list to run reduce and enlarge.

The problem is that the BG13 image doesn't process correctly. When I check them in qview, the image shows up for an instant, and then goes white. When I run the commands on each image one at a time, no problem.

I'm pretty sure the problem is in the reduce command.

Here is what I'm trying to accomplish:


cd smattson/AEB_000001_0050
**
** reduce and enlarge BG and place in a larger mosaic
**
ls *BG*.cub | sed s/.histitch.cub//> BG.lis
cat BG.lis
reduce from=\$1.histitch.cub to=\$1.S2.cub lscale=1.002 sscale=1 -batchlist=BG.lis
enlarge from=\$1.S2.cub to=\$1.S3.cub smag=1.002 lmag=1 interp=bilin -batchlist=BG.lis
handmos from=\$1.S3.cub mosaic=\$1.mos.cub samp=51 line=1 band=1 create=y nsamp=2148 nline=5500 nband=1 -batchlist=BG.lis
**
** reduce/enlarge IR11/10 and place each in larger mosaics
**
ls *IR*.cub | sed s/.histitch.cub//> IR.lis
enlarge from=\$1.histitch.cub to=\$1.S2.cub lmag=1.002 smag=1 interp=bilinear -batchlist=IR.lis
reduce from=\$1.S2.cub to=\$1.S3.cub sscale=1.002 lscale=1 -batchlist=IR.lis
handmos from=\$1.S3.cub mosaic=\$1.mos.cub samp=51 line=1 band=1 create=y nsamp=2148 nline=5500 nband=1 -batchlist=IR.lis
**

I found the same issue with the IR images, but not consistently. Sometimes it happens, sometimes at a different step. ???

Thanks for your help.
